Maze Generator

Belajar membuat generator maze otomatis di Scratch dengan konsep acak terarah dan DFS backtracking.

ğŸ¯ Tujuan Pembelajaran

  • Membuat maze otomatis di Scratch menggunakan algoritma DFS (Depth-First Search).
  • Memahami konsep acak terarah, loop, dan logika backtracking.
  • Melatih logika berpikir dan pemrograman visual dengan blok Scratch.

ğŸ§  Konsep Dasar

  • Acak Terarah: memilih jalur secara acak tapi tetap membentuk maze yang masuk akal.
  • DFS Backtracking: menjelajah sel, mundur jika buntu, hingga semua sel dikunjungi.
  • Perfect Maze: hanya ada satu jalur dari start ke finish, tanpa loop.

ğŸ§° Persiapan Proyek

  • Tambahkan Extension Pen.
  • Buat sprite Generator untuk menggambar maze.
  • Buat sprite Pemain untuk uji coba navigasi.
  • Buat variabel: cellSize, cols, rows.
  • Buat list: stack, visited.

ğŸ” Algoritma DFS Backtracking

  1. Mulai dari sel awal (misalnya 0,0), tandai sebagai dikunjungi dan masukkan ke stack.
  2. Selama stack tidak kosong:
    • Ambil sel terakhir dari stack.
    • Cari tetangga yang belum dikunjungi.
    • Jika ada, pilih satu secara acak, hapus dinding di antara, tandai dikunjungi, dan tambahkan ke stack.
    • Jika tidak ada, hapus sel dari stack (mundur).
  3. Setelah selesai, gambar dinding berdasarkan data sel.

ğŸ§© Blok Scratch yang Digunakan

  • Events: when green flag clicked
  • Control: repeat, forever, if, repeat until
  • Operators: pick random, <, >, =, and, or
  • Variables: set, change
  • Lists: add, delete, item # of, length of
  • Pen: clear, pen down, pen up, set pen size, set pen color
  • Motion: go to x: y, move

ğŸ“š Langkah-Langkah Pembelajaran

  1. Inisialisasi variabel dan list saat green flag diklik.
  2. Set posisi awal dan tambahkan ke stack dan visited.
  3. Gunakan loop repeat until untuk menjalankan DFS.
  4. Setiap langkah, pilih tetangga acak dan hapus dinding di antara sel.
  5. Setelah selesai, gambar dinding dengan Pen berdasarkan data sel.
  6. Tambahkan sprite Pemain dan kontrol dengan tombol panah.

ğŸ§ª Pengujian dan Peningkatan

  • Pastikan semua sel dikunjungi dan maze tidak bolong.
  • Tambahkan tombol "Generate" dan "Reset" untuk memulai ulang.
  • Ubah cellSize untuk tingkat kesulitan berbeda.
  • Tambahkan timer dan skor untuk tantangan navigasi.

âš ï¸ Kesalahan Umum

  • Nama variabel/list tidak konsisten: hindari spasi dan campur huruf besar-kecil.
  • Konversi koordinat salah: gunakan rumus yang sama di semua bagian.
  • Dinding satu arah: selalu hapus dinding di kedua sel (current dan neighbor).
  • Pen tumpang tindih: gunakan pen up/pen down dengan jarak tepat cellSize.
